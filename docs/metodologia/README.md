# Metodología — Spec-Driven Work + Work Orchestration

Material de referencia sobre la metodología que sigue este proyecto:

- **The Six Pillars of Spec-Driven Work** — Leonardo Gonzalez (2025)
- **From Spec-Driven Work to Work Orchestration** — Leonardo Gonzalez (2026)
- **`orchestration_agentic_transcript.txt`** — transcripción del video curso

## Por qué está en una carpeta separada

Estos documentos son **propiedad intelectual de sus autores** y por eso están
excluidos de git (ver `.gitignore`). Existen localmente solo como referencia
para el equipo.

Los principios derivados de esta metodología están sintetizados —y son auditables—
en [`../../AGENTS.md`](../../AGENTS.md) sección "Metodología".

## Resumen aplicado al proyecto

- **Los 6 Pilares** → cada ciclo cierra con `docs/CICLO_N_ARTEFACTOS.md`
- **Los 5 Principios de Autonomía** → AGENTS.md como memoria persistente, descomposición en unidades < 8 h, scripts especializados, métricas en cada ciclo, ruta de lectura para nuevos colaboradores
- **Patrón OpenSymphony** → reglas no negociables en `WORKFLOW.md`

## Cómo obtener los PDFs

Si eres miembro del equipo y necesitas los PDFs originales, solicítaselos al
líder del proyecto. No están en este repo público por derechos de autor.
